# Changelog

All notable changes to FindDrive will be documented in this file.

Format: `[YY.MM.NN]` — рік.місяць.порядковий_номер_релізу_за_день

---

## [24.06.59] — 2026-06-25

### Changed
- **Тільки одна точка на день**: поле ручного вводу адреси прибрано — місце зустрічі обирається виключно з чіпів точок зустрічі
- **Кнопка "Прибрати"**: якщо точка вже збережена — з'являється кнопка скидання місця для цього дня
- **`_combSaveLoc`** тепер читає дані з `window._pendingMpLoc` (встановлюється при кліку на чіп або при відкритті дня з вже збереженою точкою)
- Якщо точок немає — підказка перейти до налаштувань

## [24.06.58] — 2026-06-25

### Added
- **Інструктор — карта при виборі точки зустрічі**: при кліку на чіп точки в панелі дня з'являється OSM-карта з маркером; обрана точка підсвічується зеленим; при повторному відкритті дня карта вже показана якщо точка раніше збережена
- **Студент — фільтр по точках зустрічі**: якщо інструктор має 2+ гео-точки, над датами з'являються кнопки-фільтри; вибір точки показує тільки дні коли ця точка активна + OSM-карта з маркером

### Changed
- `renderBookingDates` рефакторинг: `allDates` збирає до 28 дат (для фільтрації), `dateGridHtml` — закрита функція; `window._bkSetMp` керує фільтром
- `_selectMpChip` тепер використовує `classList.active` замість inline style

## [24.06.57] — 2026-06-25

### Added
- **Графік роботи в бронюванні** — над датами з'явився блок із днями/годинами роботи інструктора і точками зустрічі (клікабельні посилання на Google Maps якщо є координати)

## [24.06.56] — 2026-06-25

### Added
- **Тривалість заняття в кабінеті учня** — тогл 1 год / 2 год зберігається у Firestore та застосовується при бронюванні

## [24.06.41] — 2026-06-25

### Added
- **Календар доступності для інструктора**: у кабінеті з'явилася секція "Розклад" з міні-календарем на поточний місяць. Кожна дата кольорово позначена: зелена = власне налаштування, світло-зелена = щотижневий розклад, червона = закрито
- **Редагування розкладу по датах**: клік на день відкриває панель з трьома режимами — "Відкрито" (задати власний час + місце), "За розкладом" (скинути до базового тижневого), "Закрито" (заблокувати дату)
- **Поле "Місце зустрічі"** для кожної відкритої дати: адреса/орієнтир, яка зберігається в Firestore і показується учню у формі бронювання як синій chip під датою
- **`getDateAvail(instr, dateStr)`**: єдина функція обчислення доступності — спочатку перевіряє per-date override в `availability` map, потім fallback до тижневого розкладу
- **Оновлена форма бронювання**: `renderBookingDates` тепер перебирає 90 днів і показує до 14 доступних дат (з урахуванням per-date overrides); `_pickDate` використовує часи з `getDateAvail`

### Changed
- `deleteField` з Firebase Firestore SDK тепер використовується для скидання per-date overrides
- `window._instrProfiles[id].availability` оновлюється локально після кожного збереження/скидання
